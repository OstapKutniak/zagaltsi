# Поведінка НПС у грі — рантайм-інтерпретатор (план + статус)

Хендоф-файл: якщо сесію перервано / зміна акаунта — звідси можна продовжити.
Контекст редактора нодів: див. памʼять Claude `zagaltsi-node-editor` та `studio.html` / `src/node-editor.ts`.

## Мета
Зробити, щоб нодові дерева поведінки ворогів РЕАЛЬНО виконувались у грі
(зараз `Enemy.think()` захардкоджено біжить на гравця, граф ігнорується).

## Ключові домовленості
- **1 «крок» = 1 клітинка колайдер-ґратки** (`colliderGrid` px, дефолт 48).
  Відстань у кроках = `Math.hypot(dx,dy) / cellSize` (floor-площина).
- Поведінка зберігається ПЕР-ПЕРСОНАЖА: IDB `zag_behavior_<charId>` (БД `zagaltsi`, store `kv`).
- Модель виконання (реактивна, щокадру): від кореня `[Поведінка]` йдемо по звʼязках;
  умови маршрутизують (Так/Ні і т.д.); проходимо крізь дії та «Потім»/reroute;
  **дією кадру стає НАЙГЛИБША досягнута поведінка**, до якої всі умови по шляху пройшли.
  Якщо жодної — ворог стоїть (idle). Це і дає видиму різницю: далеко — стоїть, близько — атакує.

## Кроки
- [x] **Крок 1. Інтерпретатор в `Enemy`** (`src/entities/Enemy.ts`)
      - поле `behavior: NodeGraph|null`, `cellSize`, `maxHp`; `setBehavior(g, cellSize)`.
      - `pickAction(player)` — обхід графа (root→Поведінка, умови, then_next, reroute, дії).
      - `evalCond` — `player_distance` (кроки=dist/cell), `health_below` (% hp),
        `sees_player`=Так(заглушка), `time_of_day`=День(заглушка).
      - `think()`: якщо є behavior → виконати обрану дію (run/walk/wait/melee/range),
        інакше старий хардкод. `moveToward()` винесено в хелпер.
- [x] **Крок 2. Завантаження графа у `GameScene`** (`src/scenes/GameScene.ts`)
      - при спавні ворога читати `idbGet('zag_behavior_'+charId)` і `enemy.setBehavior(g, gs)`.
- [ ] **Крок 3. Перевірка в грі** — заспавнити ворога з деревом distance→run→потім→distance→melee,
      переконатись: далеко стоїть, у радіусі біжить, впритул бʼє.
- [ ] **Крок 4 (майбутнє). Публікація поведінки** — зараз граф читається з IDB (працює лише на
      машині, де авторили). Для інших пристроїв/коопу треба класти граф у `public/studio-data`
      (разом із char-library або level), і вантажити в грі звідти, не з IDB.
- [ ] **Крок 5 (майбутнє).** range_attack — снаряди; sees_player — реальна лінія огляду;
      time_of_day — реальний час; поведінка нейтралів (нейтрали в грі ще не спавняться).

## Дотичні факти
- Вороги вже рендеряться по id (`Enemy.attachChar`, `GameScene` спавн ~ряд 333-348). 
- `Player.floorX/floorY`, `Enemy.fx/fy` — координати підлоги. `band` обмежує fy.
- Конфіг ворога: `src/config.ts` `ENEMY` (speed, attackRange, attackDepth, attackCooldown, damage, hp).
