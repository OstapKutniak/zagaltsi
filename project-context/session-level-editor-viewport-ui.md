## Архітектура того, що чіпали (де шукати)

Усе у [`src/level/editor.ts`](../src/level/editor.ts):

- **Геометрія ізо-ґратки** — спільні хелпери біля початку `initLevelEditor` (після `imgOf`):
  - `floorPts(cx,cy,w=1,h=1)` — підлогова клітинка (h) → 4 екранні точки. Світ: `(ix*gs+iy*k, iy*k)`, `k=gs*√½`.
  - `wallPts(cx,cy)` — стіна (v), ОРИГІНАЛЬНА стінова ґратка: `(ix*k, ix*k+iy*gs)`.
  - `floorCellAt(sx,sy)` — інверсія підлогової ґратки (клітинка під курсором).
  - `wallSnapCell(sx,sy)` — куди ляже вертикальний: `{cx:(fc.cx+1)*√2+fc.cy, cy:-(fc.cx+2)}` —
    дробові координати, нижня грань стіни точно на правій грані підлогової клітинки, стіна вгору на `gs`.
  - `floorRightEdge(cx,cy)` — 2 точки правої (45°) грані для білої лінії-снепа.
  - `fillStroke(pts, fill, stroke, lw)` — намалювати полігон.
- **Прев'ю інструментів** — блок `if (state.pathTool) { … }` у `draw()` (одразу перед `const lv = level()`):
  H — помаранчева клітинка; V — біла лінія правої грані + синій силует стіни; Y/erase — червоним
  ті h/v, що приберуться; enemy — 3×3 прев'ю + підсвітка наявної зони; enemyErase — біла рамка зони;
  spawn — клітинка в кольорі вибраного спавна. Прев'ю оновлюється бо `mousemove` тепер кличе `draw()`
  коли активний інструмент (гілка `else if (state.pathTool) draw();`).
- **`paintAt`** — h малює по `floorCellAt`, v малює по `wallSnapCell`, erase прибирає і h, і v у цій точці.
- **Спавн гравця — без прапорця**: позначається кольоровою підлоговою клітинкою (номер у центрі,
  вибраний яскравіший). `state.pathTool === 'spawn'` — інструмент розставляння; кнопка «＋ Спавн»
  (`addSpawn`) додає спавн і вмикає цей режим; `placeSpawnAt()` ставить по кліку (центр клітинки).
  Клік по клітинці спавна без інструмента — вибирає його. Старий drag-прапорець прибрано.
- **`pathTool`** тепер union: `'h' | 'v' | 'erase' | 'enemy' | 'enemyErase' | 'spawn'`.

[`src/ui-constructor.ts`](../src/ui-constructor.ts) — внизу слухач **Tab** (вмикає конструктор скрізь,
крім фокусу в input/select); нумерація кнопок за DOM-порядком (бокс тільки для видимих).

[`studio.html`](../studio.html) — hover-стилі (`.header:hover`, `.iconToggle*:hover`, `.libCard.empty:hover`,
`.imgCell.empty:hover`, `.npcCard:hover`, `button.light:hover`, `#lv-libSelect:hover`), клас `.btn-accent`
(білий hover) для помаранчевих кнопок, `#lv-levelToolbar button.on` (активний стан інструментів),
порядок іконок у нижньому ряду превʼю.

## Ключові факти / пастки

- Колайдер = ОДНА кумулятивна ізо-ґратка (`P(cx,cy)=(cx*gs+cy*k, cy*k)`, `k=gs*√½`). Та сама формула
  у `draw`, `paintAt`, і в грі ([`src/scenes/GameScene.ts`](../src/scenes/GameScene.ts) `walkableAt`/`getBandAtX`).
- `grid` (gs) — **per-level** поле, мігроване на `32`. Синкається `state.grid = level().grid` на кожній зміні рівня.
- У ГРІ вертикальні колайдери (`v`) поки **не впливають** на ходьбу — `walkableAt`/`getBandAtX` читають лише `h`.
  Тож зміни форми/снепу `v` суто візуальні в редакторі, гру не ламають.
- Воркфлоу: працюю з 2 компів/акаунтів → перед пушем завжди `git pull --rebase && git push`.
- Локальні видалення файлів у теці — навмисні, не «відновлювати з тривогою» (див. пам'ять).
- PowerShell тут — Windows PowerShell 5.1: для багаторядкових git-меседжів писати у файл і `git commit -F`,
  бо here-string ламається на лапках.
- Перевірка: `npx tsc --noEmit` має давати exit 0; dev-сервер `npm run dev` (port 5173), студія `/studio.html`.

## Відкрите / на перевірку очима після деплою

- V-колайдер: переконатись, що сідає чисто на праву грань і вгору, без зсуву (математику перевірено,
  але візуал на реальних рівнях ще не дивились).
- Снеп V — з округленням НЕ робиться (дробові коорди); якщо захочеться іншу висоту/грань — крутити `wallSnapCell`.
- `Level_Design/Refs/1.png`, `2.png` — лежать незакоміченими локально (за бажанням додати в git).
