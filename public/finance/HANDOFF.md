# Фінанси (клон 1money) — передача роботи / HANDOFF

Файл для продовження розробки з іншого акаунта/сесії Claude Code.
Стан на коміт `ca2ec63` (гілка `main`, усе запушено).

---

## 1. Що це і мета

Клон Android-додатка **1money** як **PWA** для iOS (Остап перейшов на айфон, там
1money немає; 6 років витрат треба зберегти + вести далі + синхрон між телефонами).

**Мета — зробити інтерфейс максимально ідентичним 1money** (Остап — дизайнер, хоче 1-в-1).

- Жива адреса: **https://ostapkutniak.github.io/zagaltsi/finance/**
- Ставиться на айфон через Safari → Поділитись → «На екран» (працює як нативний, без адресного рядка).

## 2. Де код і як деплоїться

- Код: **`public/finance/`** у репо `OstapKutniak/zagaltsi` (гілка `main`).
- ⚠️ **Важливо:** сайт збирається Vite-ом, GitHub Pages віддає `dist/`. Vite копіює
  `public/` у `dist/` як є. Тому додаток лежить саме в `public/finance/`, НЕ в корені.
  Якщо покласти в корінь — буде 404 (так і було спочатку).
- Деплой = **push у `main`** → GitHub Actions (`.github/workflows/deploy.yml`) → Pages (~1–2 хв).
- Базовий шлях скрізь `/zagaltsi/finance/` (manifest `start_url`/`scope`, реєстрація `sw.js`).
- Після зміни статики **піднімай версію кешу** `CACHE` у `sw.js` (зараз `fin-v5`),
  інакше телефон віддасть стару версію зі сховища (fetch — network-first, але краще бампати).

- `index.html` — розмітка: шапка, 4 екрани (categories/records/accounts/overview),

## 4. Бекенд — Firebase Realtime Database

**Перевикористано наявний Firebase гри** (проєкт `horugva-ff8bd`, той самий, що мультиплеєр —
`src/firebase.ts`). Конфіг вшито прямо в `app.js`. Нічого реєструвати не треба, синхрон з коробки.

> Спершу пробували Supabase — не дав створити проєкт («additional permissions»). Відмовились.

Шляхи в RTDB:
- `finance/transactions/{pushId}` = `{ date(ISO), type, account, category, amount(number), note }`
  - `type`: `expense | income | transfer | return`
  - для `transfer`: `account` = звідки, `category` = куди (рахунок призначення)
  - категорія може містити підкатегорію в дужках: `"Здоров'я (Адалімумаб)"`
- `finance/accounts` = масив `[{ name, balance, currency, uah, group }]` — з таблиці БАЛАНС 1money

> ⚠️ Якщо колись буде permission denied — у Firebase Console → Realtime Database → Rules
> дозволити шлях `finance`: `{ "rules": { "finance": { ".read": true, ".write": true } } }`
> (гра пише в `lobbies/`, тож база вже відкрита; поки працює без правок).

## 5. Імпорт (рідний формат 1money)

`app.js` авто-визначає формат CSV:
- **Рідний 1money** (`parse1money`): колонки `ДАТА,ТИП,З РАХУНКУ,НА РАХУНОК/ДО КАТЕГОРІЇ,
  КІЛЬКІСТЬ,ВАЛЮТА,КІЛЬКІСТЬ 2,ВАЛЮТА 2,ПОМІТКИ,НОТАТКИ`. Дата `ДД.ММ.РР`.
  Валюта: якщо не UAH — береться перерахунок (КІЛЬКІСТЬ 2/UAH), оригінал у нотатку.
- **Наш простий** (`parseSimple`): `date,type,account,category,amount,note`.
- Наприкінці CSV є таблиця **БАЛАНС** (`parseBalances`) → стартові залишки рахунків.
  Курси валют для конвертації балансів у грн виводяться з даних (`deriveRates`).
- **Імпорт ЗАМІНЮЄ дані** (спершу `remove(finance/transactions)`), щоб не було дублів.
  Тому будь-яка зміна моделі = попроси користувача переімпортувати той самий CSV.

Реальні дані: ~8616 транзакцій 2020–2026. CSV у користувача (`1Money_28_06_26_1.csv`).

## 6. Що вже зроблено (працює)

- **4 таби**: Рахунки · Категорії · Операції · Огляд. Стартовий — **Категорії**.
- **Категорії**: сітка кольорових кружечків + кільце в центрі; **тап по кільцю** перемикає
  Витрати ⇄ Доходи. Групування по батьківській категорії. Суми за період збігаються з 1money
  (звірено: червень витрати 57 041 / доходи 29 300 ✓).
- **Калькулятор додавання** (як 1money): тап по категорії відкриває екран; рахунок↔категорія
  угорі (дохід — навпаки), **чипи підкатегорій** з даних, арифметична клавіатура, дата, нотатки.
- **Операції**: список по днях з денними підсумками; тап → ✎/✕ (редагувати/видалити).
- **Рахунки**: баланси з таблиці БАЛАНС у своїй валюті, поділ Рахунки / Заощадження.
- **Огляд**: баланс, перемикач Витрати/Доходи, розбивка по категоріях з % .
- **Період** (тап по даті): Місяць / Рік / Весь час + стрілки.
- **Фільтр рахунків** (тап по сумі зверху): вибір рахунків, фільтрує всі екрани.
- Світла тема, SVG-іконки категорій (`STYLE` + `ICONS` в `app.js`), емодзі немає.

## 7. TODO / відомі проблеми (що фіксити далі)

1. **Аналітика по підкатегоріях** — головне з наступного. Дані вже містять підкатегорії
   (`subCat()`), треба екран/дрилдаун: напр. скільки на «Адалімумаб» за рік. Ідея: в «Огляд»
   тап по категорії → розкрити підкатегорії з сумами; або окремий екран категорії.
2. **Загальна сума «Всі рахунки»** може не збігатися з 1money точно — бо 1money рахує валютні
   рахунки за своїм внутрішнім курсом, а ми оцінюємо курс із даних (`deriveRates`).
   Гривневі рахунки збігаються точно. Валютні (€,$,zł) показуються в своїй валюті.
3. **Баланси валютних рахунків не оновлюються коректно** при нових операціях (мультивалютність
   спрощена — зберігаємо стартовий баланс з таблиці, рух у валюті не перераховуємо).
4. **Порядок категорій** — зараз сортування за сумою; у 1money фіксований власний порядок
   (його в експорті немає). Можна додати ручне впорядкування.
5. **Іконки категорій** — наближені SVG, не 1-в-1 з 1money. Дизайнер може схотіти точніші.
6. **Іконки PWA** (`icons/*.png`) — однотонні плейсхолдери, зробити нормальні.
7. **Layout сітки категорій** — кільце в центрі через CSS grid (donut-cell `grid-column:2/4;
   grid-row:2/4` + `grid-auto-flow: dense`); перевірити на різній к-сті категорій.
8. Дрібниці: екран «Огляд» без стовпчикового графіка по днях (у 1money він є).

## 10. Ключові місця в app.js (орієнтири)

- `firebaseConfig`, `TX_PATH`, `ACC_PATH` — підключення/шляхи.
- `STYLE`, `ICONS`, `catStyle()` — кольори/іконки категорій (keyword-match + fallback).
- `parentCat()` / `subCat()` — батьк. категорія / підкатегорія з назви.
- `rebuildLookups()` — будує списки категорій/підкатегорій/рахунків.
- `renderCategories()` + `donutSVG()` — головний екран з кільцем.
- `renderRecords()`, `renderAccounts()`, `renderOverview()` — інші таби.
- `openForm()` / `renderForm()` / `keyPress()` / `evalExpr()` / `saveCalc()` — калькулятор.
- `importCSV()` / `parse1money()` / `parseBalances()` / `deriveRates()` — імпорт.
- `openFilter()` / `drawFilter()` — фільтр рахунків. `openPeriod()` — період.
